# Darask Novel Engine

`scenario.txt` というテキストファイルを置くだけで動く、超シンプルなノベルゲームエンジンです。
ビルド不要・依存ライブラリなしの HTML/CSS/JS で完結し、**ブラウザ**でも、
Electron でビルドした **Windows 用 exe**(ウィンドウアプリ)でも動きます。

```
アオイ:ねえ、まだ帰らないの?
ユウタ:もうちょっとだけ。この課題を終わらせたいんだ。
夕日が教室を茜色に染めていた。
```

↑ こう書くだけでノベルゲームになります。左が名前、`:` の右がセリフ。
名前を書かない行は「地の文(ナレーション)」になります。

## 5分で試す(クイックスタート)

1. このフォルダで簡易サーバーを立てます(Python があれば1行):

   ```
   python -m http.server 8000
   ```

2. ブラウザで `http://localhost:8000` を開く → サンプルがそのまま遊べます。
3. `scenario.txt` をテキストエディタで書き換えて、ブラウザを再読み込みするだけ。

> 画像や音声が無くてもOK。ファイルが見つからないときは、名前入りの
> プレースホルダ画像が自動生成され、音声は無視されます。
> **アセット0個でもサンプルは最後まで動きます。**

## 遊び方・起動方法

### A. ブラウザで動かす(おすすめ)

`scenario.txt` を自動で読み込むにはローカルサーバー経由で開く必要があります。
どれか1つの方法でサーバーを立てて、表示された URL を開いてください。

| 方法 | コマンド |
|---|---|
| Python(標準で入っていることが多い) | `python -m http.server 8000` |
| Node.js | `npx serve` または `npx http-server` |
| VS Code | 拡張機能「Live Server」で `index.html` を右クリック→ Open with Live Server |

### B. `index.html` をダブルクリックで直接開く

サーバーを立てずに開くと、ブラウザのセキュリティ制限で `scenario.txt` を
自動読み込みできません。この場合は**ファイル選択画面**が出るので、
`scenario.txt`(分岐で複数ファイルを使うならまとめて)を選ぶか、
ウィンドウにドラッグ&ドロップしてください。

### C. exe(ウィンドウアプリ)で動かす

配布された exe は**ダブルクリックするだけ**でウィンドウが開きます。
さらに、**exe と同じフォルダに `scenario.txt` や `bg/` などのフォルダを置くと、
同梱シナリオより優先して読み込みます**。exe を配り、テキストと画像を
差し替えるだけで別のゲームにできます。

自分で exe を作る方法は [ビルド方法](#ビルド方法exe-を作る) を参照。

## テキストファイルの書き方

シナリオは `scenario.txt`(UTF-8)から始まります。1行が1つの動作です。

### 基本(セリフ・地の文・コメント)

| 書き方 | 意味 |
|---|---|
| `名前:セリフ` | キャラのセリフ。コロンは半角 `:` / 全角 `：` どちらでもOK。名前は**12文字以内・空白なし** |
| コロンの無い行 | 地の文(ナレーション) |
| 空行 | 無視(見やすさのために自由に空けてOK) |
| `#` `//` `;` で始まる行 | コメント(無視) |

```
# これはコメント。実行されません
ユウタ:おはよう、アオイ。
アオイ：おはよ。今日はいい天気だね。      ← 全角コロンでもOK
教室の窓から、朝の光が差し込んでいた。      ← 地の文
```

### @コマンド一覧

行頭が `@` の行は演出コマンドです。

| コマンド | 説明 |
|---|---|
| `@title タイトル名` | ゲームのタイトルを設定(タイトル画面に表示) |
| `@bg ファイル名` | 背景画像を変更(`bg/` フォルダから) |
| `@chara 名前 [位置] [表情]` | 立ち絵を表示(`chara/` フォルダ)。`@show` も同義 |
| `@face 名前 表情` | 立ち絵はそのままで表情だけ差し替え |
| `@hide 名前` / `@hide all` | 立ち絵を退場(`all` で全員) |
| `@cg ファイル名` | 一枚絵(CG)を全画面表示。CGギャラリーに自動登録 |
| `@cg off` | CG を消す |
| `@bgm ファイル名` | BGM をループ再生(`bgm/` フォルダ) |
| `@bgm stop` | BGM を停止 |
| `@se ファイル名` | 効果音を1回再生(`se/` フォルダ) |
| `@scene シーン名` | シーンの区切り。到達すると「シーン鑑賞」で再生可能に |
| `@wait ミリ秒` | 少し待つ(例:`@wait 800`。スキップ中は無視) |
| `@set フラグ操作` | フラグを直接操作(例:`@set 好感度+1`、`@set 出会った=true`) |
| `@jump ファイル.txt` | 別のシナリオファイルへ移動 |
| `@if 条件 ファイル.txt` | 条件を満たすときだけ別ファイルへ移動 |
| `@end` | 物語を終了してタイトル画面へ戻る |

### 立ち絵と表情差分

立ち絵は**背景とは別のレイヤー**で表示され、`chara/` フォルダから読み込みます。
表情は「**ベース画像 + 顔の差分**」を重ねて表現します(差分は透過PNG推奨)。

ファイルの命名規約(拡張子 `.png` は自動で付きます):

```
chara/アオイ.png        ← ベース(体・輪郭)
chara/アオイ_笑顔.png    ← 表情差分「笑顔」(顔だけ描いた透過PNG)
chara/アオイ_泣き.png    ← 表情差分「泣き」
```

書き方:

```
@chara アオイ right 笑顔   ← 右に「笑顔」で登場
@chara ユウタ left         ← 左に登場(表情省略=ベースのみ)
アオイ:やっと来た!
@face アオイ 呆れ           ← 立ち絵はそのまま、表情だけ変更
アオイ:遅刻ギリギリだよ?
@hide アオイ               ← アオイだけ退場
@hide all                 ← 全員退場
```

- 位置は `left` / `center` / `right`(`左` `中` `右` も可)。**省略すると今の位置を維持**。
- 喋っているキャラの立ち絵が明るくなり、それ以外は自動で少し暗くなります。
- 立ち絵はセーブや「シーン鑑賞」にも保存され、ロード時に位置・表情ごと復元されます。

### 選択肢

`選択肢` とだけ書いた行の**次の行から**が選択肢です。空行または `@` コマンドで
ブロックが終わります。プレイ中は**クリック**または**数字キー 1〜9** で選べます。

```
どうする?

選択肢
話しかける
そっと見守る
帰る
```

各選択肢に `->` を付けると「効果」を持たせられます(カンマ区切りで複数可):

```
選択肢
手伝ってもらう -> 好感度+1
自分でやると断る -> 意地=true
アオイを推薦する -> 好感度+1, route_b.txt
くじ引きにする
```

| 効果の書き方 | 意味 |
|---|---|
| `フラグ名+1` / `フラグ名-1` | 数値フラグを増減 |
| `フラグ名=true` / `フラグ名=false` | 真偽値フラグを設定 |
| `フラグ名=5` | 数値を直接代入 |
| `ファイル名.txt` | そのシナリオファイルへ続きを移す(分岐) |

### フラグと分岐(複数ファイル)

選択肢や `@set` で立てたフラグを、`@if` で判定して別ファイルへ飛ばせます。
条件には `>=` `<=` `>` `<` `==` `!=` が使えます。

```
# route_a.txt の末尾など
@if 好感度>=3 ending_true.txt      ← 好感度が3以上なら真エンドへ
@if 意地==true ending_bad.txt      ← 意地フラグが true ならバッドエンドへ
@jump ending_normal.txt            ← どれにも当てはまらなければ通常エンドへ
```

シナリオを複数ファイルに分けることで、ルート分岐やエンディング分岐を作れます。
`scenario.txt` が最初に読まれるファイル(エントリーポイント)です。

### 完全なサンプル

これだけで動く最小のノベルです(`scenario.txt` に貼り付けてサーバー経由で開くだけ)。

```
@title はじめてのノベル

@scene プロローグ
@bg classroom.jpg
@bgm main.mp3

放課後の教室。夕日が机を赤く染めている。

@chara アオイ center 笑顔
アオイ:やっと終わったね、掃除当番!
@face アオイ 普通
アオイ:このあと、ちょっと寄り道してかない?

選択肢
一緒に帰る -> 好感度+1, route_together.txt
今日はやめておく -> route_alone.txt
```

`route_together.txt`:

```
@scene 寄り道
@bg street.jpg
アオイ:わたし、この道の夕焼けが好きなんだ。
@face アオイ 照れ
アオイ:……また、一緒に帰ってくれる?

@if 好感度>=1 ending_good.txt
@jump ending_normal.txt
```

`ending_good.txt`:

```
@scene エンディング
@bg sunset.jpg
@bgm ending.mp3
二人の影が、長く伸びていた。
―― HAPPY END ――
@end
```

## アセット(画像・音声)の入れ方

各フォルダに置いて、シナリオ内ではファイル名だけで参照します。

| 種類 | フォルダ | シナリオでの参照 | 推奨形式 |
|---|---|---|---|
| 背景 | `bg/` | `@bg classroom.jpg` | jpg / png(16:9 推奨) |
| 立ち絵 | `chara/` | `@chara アオイ ...` → `chara/アオイ.png` | 透過 png(縦長) |
| 表情差分 | `chara/` | `@face アオイ 笑顔` → `chara/アオイ_笑顔.png` | 透過 png |
| イベントCG | `cg/` | `@cg event01.jpg` | jpg / png(16:9 推奨) |
| BGM | `bgm/` | `@bgm main.mp3` | mp3 / ogg / wav |
| 効果音 | `se/` | `@se chime.mp3` | mp3 / ogg / wav |

- **無いファイルはエラーになりません。** 画像はファイル名入りのプレースホルダ、
  音声は無音になります。まず文章だけ書いて、後からアセットを足せます。
- 画面比率は **16:9** です。背景・CGはこの比率に合わせると綺麗に表示されます。
- ブラウザの自動再生制限により、BGM は最初のクリック以降に鳴ることがあります。

| キー | 機能 |
|---|---|
| Enter / Space / クリック | 読み進める |
| Ctrl(押しっぱなし) | スキップ |
| A | オートモード切替 |
| H / 右クリック | メッセージウィンドウを隠す |
| B / ホイール上 | バックログ(履歴) |
| F | フルスクリーン |
| S | クイックセーブ |
| L | クイックロード |

## ビルド方法(exe を作る)

Windows 用のポータブル exe(インストール不要の単体 exe)を作れます。

### 前提

- [Node.js](https://nodejs.org/)(18 以上推奨)がインストールされていること。
  確認: `node -v`

### 手順

```
npm install        # 初回だけ。依存(Electron など)を取得
npm run start      # ビルドせずにそのままウィンドウで起動して確認
npm run dist       # dist/ にポータブル exe を生成
```

`npm run dist` が終わると `dist/DaraskNovelEngine 1.0.0.exe` ができます。
これ1つを配布すればOK(約 70MB。Electron ランタイム同梱のため)。

### 配布したあとにシナリオを差し替える

exe と**同じフォルダ**に `scenario.txt` や `bg/` `chara/` `bgm/` などを置くと、
そちらが優先して読み込まれます。exe 本体を作り直さずに中身を差し替えられます。

### exe のアイコンを変える

exe のアイコンは `build/icon.ico`(同梱のデフォルトあり)です。
**自分の `.ico` で上書きするだけ**で差し替えられ、次回 `npm run dist` から反映されます。

- 形式: Windows の `.ico`。**256×256 を含むマルチサイズ**推奨(16/32/48/256 など)。
- 参照先は `package.json` の `build.win.icon` で指定しています(既定 `build/icon.ico`)。
- `npm run start`(開発実行)時のウィンドウ/タスクバーには `build/icon.png` を使います。
  合わせて差し替えると見た目が揃います。
- PNG しか手元に無い場合は、オンラインの ico 変換や ImageMagick
  (`magick icon.png -define icon:auto-resize=256,48,32,16 build/icon.ico`)で作成できます。

### バージョンや productName を変える

`package.json` の `version` / `build.productName` を編集してください。

## CI/CD(Web版の自動デプロイ)

`main` に push するたびに Web 版を Cloudflare Pages へ自動デプロイする
ワークフローを同梱しています。

[.github/workflows/deploy-pages.yml](.github/workflows/deploy-pages.yml)

事前準備:

1. Cloudflare ダッシュボード → **Workers & Pages** → Pages プロジェクト
   `darask-novel-engine` を作成(Direct Upload)。
2. GitHub リポジトリの **Settings → Secrets and variables → Actions** に登録:
   - `CLOUDFLARE_API_TOKEN`(Cloudflare Pages 編集権限のトークン)
   - `CLOUDFLARE_ACCOUNT_ID`

> GitHub Actions を使わず、Cloudflare Pages の「Git 連携」でこのリポジトリを
> 直接つないでもOKです(ビルドコマンドなし・出力ディレクトリ `/`)。

> exe はエンジン自体には不要なため、自動ビルドは用意していません。
> 必要なときだけ手元で `npm run dist` を実行してください
> ([ビルド方法](#ビルド方法exe-を作る))。

## 困ったとき(トラブルシューティング)

**Q. `index.html` を開いてもサンプルが始まらない**
A. ダブルクリックで直接開くとシナリオを自動読み込みできません。
[ローカルサーバー経由](#a-ブラウザで動かすおすすめ)で開くか、出てきた
ファイル選択画面から `scenario.txt` を選んでください。

**Q. 画像・音声が表示/再生されない**
A. ファイル名・拡張子・置き場所(`bg/` `chara/` など)を確認してください。
無い場合はプレースホルダ/無音になります。BGM はブラウザの制限で
最初のクリック後に鳴ることがあります。

**Q. 立ち絵の表情が変わらない**
A. `chara/名前_表情.png` というファイル名になっているか確認してください
(例:`@face アオイ 笑顔` は `chara/アオイ_笑顔.png` を探します)。

**Q. Windows で `npm run dist` が「Cannot create symbolic link」で失敗する**
A. electron-builder が署名ツールを展開する際にシンボリックリンクを作るためです。
**設定 → 開発者向け → 開発者モード**を有効にするか、管理者権限のターミナルで
実行してください。

**Q. セーブや設定を初期化したい**
A. ブラウザの開発者ツール → Application → Local Storage で `dne_` で始まる
項目を削除してください(exe の場合はアプリのデータフォルダ内)。
